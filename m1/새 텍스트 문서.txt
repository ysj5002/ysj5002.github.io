

	모바일 웹(앱)에 대한 제작 및 개발 단계
		기획 : UI/UX 계획수립, 사용자 리서치, UI/UX 요구분석, UI/UX 콘셉트 기획
		분석 및 설계 : UI/UX 환경 분석, UI 아키텍쳐 설계
		디자인 : UI 디자인
		구현 : UI 구현, UI테스트
		완료 : UI/UX 가이드 제작

	★ 기획 과정 ★
	UI/UX 계획수립 : 개발 제안 내용에 대하여 개발계획일정수립, 비용투입계획, 인력투입계획, 개발규모, 주안점 등을
			바탕으로 계획을 수립 -> WBS
	사용자(Persona) : 일반 관공서 모바일 웹(앱) 이용자 -> 주사용자, 해당 관공서의 종사자 -> 부사용자
		-> 사용자 설문 및 인터뷰 결과
	사용흐름 : 주 사용자의 사용 시나리오 분석
		-> 사용자 시나리오
	요구 분석 : 클라이언트의 요구사항(Needs)에 대하여 어떻게 충족시켜 줄 것인지에 대한 분석
		-> 니즈(Needs List)
	콘셉트 기획 : 어떠한 느낌과 기분을 느끼게 할 것인지에 대하여 색, 레이아웃, 아이콘, 형태, 폰트 등을 정하는 것.
		-> 무드보드, 스케치, 페이퍼 프로토타입

	★분석 과정 ★
	UX 트렌드 분석 : 디자인 트렌드 분석 -> 현재 모바일 웹(앱)의 디자인에 대한 트렌드 분석
		-> 디자인 제작 가이드
	트렌드 기술 분석 : 구현 트렌드 분석 -> 현재 모바일 웹(앱)의 제작 및 구현 기술 분석
		-> UI 구현 가이드
	사용자의 라이프스타일 분석 : 사용자가 해당 콘텐츠를 접했을 때가 언제이며, 사용횟수가 많은 경우를 삶 속에서 찾아 분석하는 과정
		-> 라이프스타일 AIO

	★ 설계 ★
	시스템과 전체에 대한 설계 : WBS와 NEEDS LIST에 따라 전체에 대한 정보 구조를 설계
		-> 사이트맵 또는 정보구조도
	상세 또는 각 화면 설계 : 메인화면부터 서브화면, 상세페이지까지 각 화면을 상세히 기술하고 설계
		-> 화면 설계서, 스토리보드
	기타 및 인터랙션 설계 : 기능적 설계, 변화 설계, DB 설계 등이 여기에 속한다.
		-> UML 구조도, 기능 정의서, 인터렉션 페이퍼, DB설계서

	★ 디자인 ★
	스타일가이드 제작 : 진행 중인 프로젝트의 각 시안에 들어갈 내용(색, 아이콘, 레이아웃, 타이포)을 하나의 문서로 만듦
		-> 스타일가이드
	디자인 템플릿 : 포토샵과 같은 프로그램을 이용해 각 화면을 디자인하는 것으로 메인(시안) 템플릿, 서브(시안) 템플릿을 제작
		-> 디자인 PSD 시안
	스키마 제작 : 색, 아이콘, 레이아웃, 타이포 등에 대한 상세한 정의와 명명과 설명, 사용방법이 있는 문서
		-> 컬러 스키마(Color class), 아이콘 스키마(Icon class), 레이아웃 스키마(html layout), 타이포 스키마(font-face class)

	★ 구현 ★
	화면구현 : html+css와 준비된 텍스트, 이미지, 동영상, 사운드 소스를 이용하여 화면에 출력될 수 있도록 구현
	인터렉션 구현 : css+javascript+jquery를 이용해 변화 요소를 구현  ex) 애니메이션 효과, 폼 요소의 구현
	기능구현 : db 기능이나 파일 기능이 있는 요소를 입출력 할 수 있도록 구현   ex) 게시판, 회원가입 등
	UI 테스트 : 통합테스트와 단위테스트를 통틀어 진행하는 테스트로 디자인을 테스트할 경우는 리뷰(review)라고 함
		통합테스트는 전체적인 내용과 기능을 테스트하게됨 -> 테스트계획서, 통합 테스트 체크리스트
		단위테스트는 세부적인 내용과 기능을 테스트하게됨 -> 테스트 계획서, 단위 테스트 체크리스트
		리뷰테스트는 디자인 위주로 하는 테스트를 말함 -> 리뷰 계획서, 리뷰 체크리스트
		수정, 보완 및 최종 테스트 -> 각 테스트를 거치고 오류에 대한 수정과 부족한 내용을 보완하는 단계

	★ 완료 ★
	UI 가이드 제작 : 클라이언트의 납품 및 최종보고에 앞서 설명할 내용과 사용자 메뉴얼, 관리자 메뉴얼, 개발 가이드 등을 제작
	최종보고 : 계약에 대하여 최종적으로 완료될 수 있도록 클라이언트의 확인을 받고, 프로젝트를 종결하는 단계
	유지보수 : 각 단계의 산출물을 근거로 별도의 유지보수 단계를 진행할지를 계약하고, 서비스를 해주는 단계
	사후관리 : 진행했던 프로젝트에 대하여 이상이나 부족한 부분을 보충하고, 수정하는 단계
